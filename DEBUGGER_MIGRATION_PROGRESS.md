Build verified (macOS Debug).
3. Disassembly Routine
**DONE** — Refactored monolithic `Processor::Disassemble()` + `DisassembleNextOpcode()` into Geargrafx-style split methods: `DisassembleNextOPCode()`, `PopulateDisassemblerRecord()`, `InvalidateOverlappingRecords()`, `DisassembleAhead(int)`, `DisassembleAhead(u16,int,int)`, `CheckBreakpoints()`. All new `GS_Disassembler_Record` fields populated: `segment` (address-range detection: ROM/RAM/BIOS), `subroutine` (CALL nn, CALL cc,nn, RST xx), `jump_bank` (via `Memory::GetBank()`), `has_operand_address`/`operand_address` (type 3 operands), `irq` (via new `m_debug_next_irq` member, set to 2 on NMI, 3 on INT). Added `m_debug_next_irq` field (s32) initialized to 1 in constructor and `Reset()`. `InvalidateOverlappingRecords` scans 7 bytes backward and forward to clear stale overlapping records. `DisassembleAhead` recursively follows jump targets (depth-limited to 3) and stops at unconditional control flow (RET, JP nn, JR, HALT, RETI, RETN). Renamed all call sites from `DisassembleNextOpcode` to `DisassembleNextOPCode` (Processor.cpp RunFor, GearsystemCore.cpp). Uses `Memory::Retrieve()` (not `Read()`) to avoid triggering memory breakpoints during disassembly. Build verified clean (macOS Debug, zero warnings).
